一、以工作日的一天作为场景演示

本文档按触发方式重新分类。机器人端负责唤醒、感知、屏幕展示、动作、灯光和语音播放；OpenClaw 侧负责理解事件、读取飞书/外部数据、执行业务动作，并返回结构化响应。

## 1. 主动交互任务

主动交互任务由用户当场发起，OpenClaw 只有收到用户输入后才执行。

### 1.1 早上10:00-日程建立

- 触发方式：用户唤醒机器人后发出自然语言指令。
- 用户指令：hey, 小O
- 机器人屏幕展示：转头，【眨眼动态表情】
- 机器人语音回复：我在呢，请问有什么可以帮您？
- 用户指令：明天上午10点到11点的项目会，帮我建一个飞书日程，邀请xxx、xxx、xxx参会。
- 机器人屏幕展示：【思考动态表情】
- OpenClaw 业务动作：
  - 解析日程标题、日期、开始和结束时间。
  - 解析参会人并查询飞书通讯录。
  - 创建飞书日程并邀请参会人。
  - 当时间、标题、参会人缺失或有歧义时先追问，不直接创建。
- 机器人语音回复：好的，已经为你创建好日历，并把参会人员拉进日程里啦。
- 用户二次指令：好的谢谢！
- 机器人屏幕展示：【害羞表情】，灯光闪烁

## 2. 被动触发任务

被动触发任务不依赖用户当场发起。它们需要调度器、机器人感知服务或外部事件先产生结构化事件，再交给 OpenClaw 处理。

### 2.1 早上9:30-定时日程管理及播报

- 触发方式：系统定时触发，例如每天工作日上午9:30触发 `daily_briefing_triggered`。
- 触发条件：到达配置的日程播报时间。
- OpenClaw 业务动作：
  - 查询今日日程。
  - 查询上周或近期工作日日程。
  - 统计内部会议、客户接待、外出活动、深度工作等分类。
  - 生成工作复盘和今日日程重点提醒。
- 机器人屏幕展示：【微笑】转头，灯光闪烁
- 机器人语音回复：早上好！今天是6月1日（周一），上周工作已自动复盘：上周累计工作日程共 25 项：其中内部会议占比50%、客户接待占比20%、外出活动占比5%。本周会议占用时间偏高，深度工作时间偏少。建议本周预留整块时间用于项目攻坚与深度工作~
  同时为您汇总今日日程：今日共有2场内部会议、1场外部接待、1个外出。重点提醒：10点月度汇报请提前备好汇报资料哦~14点xxx公司来访接待，会议室在中1；16:00有一个外出，请不要迟到哦~

### 2.2 早上11:00-会议会前提醒

- 触发方式：系统扫描飞书日历后触发 `meeting_starting_soon`。
- 触发条件：检测到距离“项目会”还有 N 分钟开始，例如5分钟或10分钟，具体取决于调度器配置。
- OpenClaw 业务动作：
  - 读取会议标题、开始时间、地点、会议室、参会人或会议通知目标。
  - 生成会前提醒。
  - 将当前会议写入短期上下文，供后续“我会晚到，帮我通知参会人”这类省略对象的指令使用。
- 机器人展示：头部转向用户，【眨眼动态表情】，灯光闪烁
- 机器人回复：打扰一下，11:00项目会议还有五分钟将在中1会议室开始，请提前准备参会。

### 2.3 下午14:40-日常外勤出行规划

- 触发方式：系统扫描飞书日历后触发 `outdoor_event_detected`。
- 触发条件：检测到用户当日有外勤、客户拜访、园区拜访等日程，并到达配置的提前提醒时间。
- OpenClaw 业务动作：
  - 从日程标题、地点、备注中提取目的地。
  - 读取用户默认出发地和出行偏好。
  - 查询路线耗时。
  - 根据日程开始时间、路线耗时和提前到达缓冲时间计算建议出发时间。
  - 路线不可用时降级为提醒用户预留充足时间，不编造路线。
- 机器人屏幕展示：【眨眼微笑】
- 机器人语音回复：哈喽，您下午16:00有外出行程，目的地是xxxxx，从公司出发预计驾车路程30分钟，建议15:20出发，可提前10分钟到达目的地。
- 用户指令：好的，我知道了。
- 机器人展示：【点头】

### 2.4 下午15:00-久坐关怀情绪陪伴

- 触发方式：机器人端或感知服务触发 `sedentary_detected`。
- 触发条件：感知服务判断用户已连续久坐，例如20到30分钟，并提供置信度。
- OpenClaw 业务边界：
  - 不负责每隔20分钟拍照识别这类硬件感知逻辑。
  - 负责接收结构化久坐事件，判断是否适合打扰用户。
  - 避开会议中、通话中、刚提醒过等不适合打扰的场景。
  - 可结合近期日程顺带提醒即将发生的事项。
- 机器人屏幕展示：【舒缓轻松表情】，左右转头，灯光闪烁
- 机器人语音回复：我观察到您下午已经连续久坐工作20到30分钟啦，身体需要短暂放松。建议起身活动拉伸一下、眺望远方舒缓眼睛疲劳。
- 机器人屏幕展示：【微笑】
- 机器人语音回复：工作虽然辛苦，但也要劳逸结合~给您分享趣味小段子，希望您放松一刻、缓解疲惫哦。

### 2.5 下午18:30-异地出差出行建议

- 触发方式：系统扫描飞书日历后触发 `business_trip_tomorrow_detected`。
- 触发条件：检测到第二天有出差、差旅、航班、高铁、酒店等相关日程，并到达配置的前一日提醒时间。
- OpenClaw 业务动作：
  - 从日程标题、地点、备注中提取目的地城市。
  - 查询目的地天气。
  - 根据天气、温差、日程备注生成携带和着装建议。
  - 天气不可用时降级为出差事项提醒，不编造天气。
- 机器人屏幕展示：转头【思考动态表情】
- 机器人语音回复：哈喽，准备下班咯~提醒您明天是6月2日（周二），明早9点将前往上海出差，上海明日20℃，有60%概率降雨，与深圳温差较大，记得增添外套，带好雨具。同时提醒您记得携带身份证、充电器、出差办公资料。
- 用户指令：好的，我知道了，有什么别的事项需要我注意的吗？
- 机器人展示：点头，灯光闪烁，【开心动态表情】
- 机器人语音回复：查询到日程备注栏写着建议着正装，建议您带上一套，祝您一路平安！回来后见~

## 3. 混合任务：被动提醒后的主动二次操作

混合任务先由系统被动提醒，再由用户主动提出二次操作。二次操作通常依赖上一轮提醒留下的短期上下文。

### 3.1 会议迟到通知

- 第一段触发方式：`meeting_starting_soon` 被动触发会前提醒。
- 第二段触发方式：用户在会前提醒后主动发出迟到通知指令。
- 用户指令：我这里还有个电话会议没结束，帮我跟参会人员说一下，晚五分钟到。
- OpenClaw 业务动作：
  - 从当前上下文识别“刚刚提醒的会议”。
  - 找到会议群或参会人通知目标。
  - 通过飞书发送迟到通知。
  - 若没有会议上下文或通知目标，先追问，不发送消息。
- 机器人展示：【点头】
- 机器人语音回复：没问题，我已经通过飞书会议群，替您通知参会人员将晚五分钟到场，请大家稍作等待。
- 机器人通过飞书发送信息给参会人员。话术：“不好意思各位，xxx将稍晚五分钟参会，请稍等一下哦。”

### 3.2 日程播报的触摸入口

- 触发方式：用户摸头触发 `head_touch`。
- 说明：这是用户主动触发的日程播报入口，不等同于定时播报。定时播报属于被动触发任务，摸头播报属于主动交互任务。
- 用户指令：【摸摸头】
- OpenClaw 业务动作：复用日程播报能力，查询并生成今日日程和近期复盘。

## 4. 被动触发任务的运行前提

在真实环境中，仅仅“OpenClaw 正常运行”还不够。被动触发任务需要满足以下条件：

- `work-assistant` 插件已安装并加载。
- 插件调度器已显式启用，且 `startIntervalLoop` 已开启。
- 当前实现中，插件调度器会在 OpenClaw gateway 进程内扫描飞书日历，把到期触发事件交给内部 `assistant.handleEvent(event)` 并生成 `StructuredResponse`。如果启用了 `scheduler.agentDispatch`，调度器会把该响应包成 `openclaw.stackchan.event.v1` 的 Xiaopai 渲染 envelope，并通过 OpenClaw session workflow 入队一次 agent/LLM turn；agent 处理后的回复再通过 `xiaopaiControl.execute` 或 Xiaopai fallback 入队语音指令。机器人当前不在线时，只要 OpenClaw agent turn 和后续 Xiaopai 命令正常入队，就视为链路成功。
- 飞书日历读取权限和 `lark-cli` 用户身份配置可用。当前实现默认用 `larkIdentity: "user"`，因此需要本机用户 OAuth 授权，不是只配置 bot app secret 就够。
- 最小可运行权限：`contact:user:search`、`calendar:calendar.event:read`、`calendar:calendar.event:create`、`calendar:calendar.event:update`。
- 若需求包含忙闲查询、推荐会议时间或会议室排期，需要额外授权 `calendar:calendar.free_busy:read`。
- 若需求包含“迟到通知/通知参会人”等飞书 IM 发送，需要用户身份授权 `im:message.send_as_user` 和 `im:message`；如果改成 bot 身份发送，则需要 `im:message:send_as_bot` 且 bot 必须能访问目标会话。
- 排查命令：
  - `lark-cli auth status --verify`
  - `lark-cli auth check --scope "contact:user:search calendar:calendar.event:read calendar:calendar.event:create calendar:calendar.event:update calendar:calendar.free_busy:read im:message.send_as_user im:message"`
- 对应调度规则已启用：
  - `dailyBriefing` 用于定时日程播报。
  - `meetingStartingSoon` 用于会前提醒。
  - `outdoorEvent` 用于外勤出行提醒。
  - `businessTripTomorrow` 用于次日出差提醒。
- 飞书日程标题、地点或备注中包含可匹配的关键词，例如“会议”“同步”“客户”“拜访”“园区”“外出”“出差”“航班”“高铁”等。
- 对于迟到通知，飞书日程或调度事件中需要有可用的会议群 `chat_id` 或参会人 ID；否则只能生成追问，不能直接发消息。
- 对于路线和天气，当前实现只有 dry-run 模拟数据；真实路线和真实天气服务需要后续接入 provider。未接入时会降级提醒，不会编造数据。
